# 이달아 (이달의 아트) — 사진으로 만드는 이달의 네일 아트 (디자인 문서)

> 서비스명: **이달아** — "이달의 아트"의 줄임말. 매달 새로운 네일 아트를 만들어내는 감각을 담는다. (저장소 코드명: nailspo)

날짜: 2026-07-07
상태: 사용자 승인 대기

## 한 줄 요약

맘에 드는 영감 사진(풍경, 무드보드, 사물 등)을 **1~3장** 올리면, 그 사진들의 색감·질감·분위기를 종합해 어울리는 네일 디자인을 AI가 생성해주는 모바일 웹앱. 결과는 실제 네일 디자이너들이 인스타에 올리는 형식 그대로 — **생성된 네일 사진 위에 영감 사진들이 작게 얹힌 콜라주** 한 장으로 저장된다 (`ref/` 폴더의 참고 이미지 11장이 이 형식). **타겟은 매달 "이달의 아트"를 여러 개 구상해야 하는 네일 디자이너**로, 영감 사진 몇 장에서 새 디자인 시안을 빠르게 뽑아 매달 아이디어 발상의 출발점으로 쓴다.

핵심 경험은 **"사진을 더할수록 디자인이 진화한다"**: 빨간 사진 한 장으로 만들었을 때와, 거기에 초록 사진을 더해 다시 만들었을 때 결과가 달라진다 (사과·싱그러움 같은 무드가 더해짐). 디자이너는 사진을 추가·교체해가며 시안을 발전시킨다.

## 배경과 목적

- 네일아티스트들은 영감 사진 여러 장(물결+꽃, 바다+진주+조개, 목걸이+자동차 등 이질적 조합)을 보고 네일 디자인을 만든 뒤, 완성 네일 사진 위에 영감 사진들을 작게 얹은 콜라주로 인스타/틱톡에 올린다. **이 저장소의 `ref/` 폴더에 실제 사례 11장이 있다** — 결과물 형식과 네일 미감의 기준.
- 이달아는 이 수작업 과정(영감 수집 → 디자인 구상 → 콜라주 제작)을 통째로 자동화한다.
- **핵심 타겟: 네일 디자이너.** 이들은 매달 "이달의 아트"를 여러 개씩 새로 구상해야 하는 창작 압박이 있다. 이달아는 영감 사진 1~3장을 넣으면 어울리는 네일 디자인 시안을 즉석에서 뽑아, 디자이너의 매달 아이디어 발상을 돕는 도구가 된다.
- 즉, 이달아는 "손님용 시안 뽑기"가 아니라 **디자이너의 창작 보조 도구**다. 모든 화면·문구·기능 판단은 디자이너 관점을 우선한다.
- 사용자(개발자)의 목적: 한국 시장 대상 소규모 수익형 웹 유틸리티. 이번 단계는 수요 검증 MVP.
- **핵심 전제: 네일 트렌드는 계속 바뀐다.** 계절·시즌마다 유행하는 색감, 파츠, 스타일(예: 글레이즈드, 마그네틱, 치크 네일 등)이 달라지므로, 생성 결과가 항상 최신 트렌드를 따라가야 서비스 가치가 유지된다. 트렌드에 뒤처진 촌스러운 결과물은 재방문을 막는다.

## 확정된 결정 사항

| 항목 | 결정 |
|------|------|
| 결과물 | **콜라주 한 장** = AI 생성 네일 이미지(실사 느낌) 위에 입력한 영감 사진들이 작은 사각형으로 얹힌 형태 (`ref/` 형식) + 무드 키워드/색상 팔레트 텍스트 |

| 진화 방식 | 사진을 트레이에 담아 **"만들기"를 누를 때 담긴 사진 전체를 종합해 한 번에 생성**. 결과를 본 뒤 사진을 추가/교체하고 다시 만들면 새 세트 기준으로 재생성 (이것이 "진화") |
| 요금 | 무료, 1인당 하루 3회 제한 + 서비스 전체 하루 총량 제한 |
| 플랫폼 | 모바일 우선 웹앱 (설치 불필요, 링크로 접근). **향후 iOS/Android 네이티브 앱 배포 예정** — 이번 MVP는 웹앱으로 검증 |
| 디자인 톤 | 매우 트렌디하고 힙한 UI/UX (디자이너 타겟에 걸맞은 감각). 아래 "UI/UX 디자인 원칙" 참고 |
| 회원가입/DB | 없음 (MVP에서는 불필요) |
| 접근 방식 | 방법 A: Gemini 이미지 모델 1회 호출로 이미지+설명 동시 생성 |

## UI/UX 디자인 원칙

핵심 타겟이 **미적 기준이 높은 네일 디자이너**이므로, 첫인상에서 "감각 있다"는 느낌을 줘야 재사용·입소문이 난다. 흔한 템플릿 느낌은 금물.

- **트렌디·힙한 무드**: 요즘 감성의 편집숍/패션 앱 같은 세련된 룩. 과감한 여백, 큰 타이포, 절제된 포인트 컬러.
- **콘텐츠(결과 이미지)가 주인공**: UI는 최대한 비켜서고, 생성된 네일 이미지가 화면에서 가장 돋보이게 한다.
- **감각적인 모션**: 로딩·전환에 부드럽고 세련된 애니메이션. 단, 과하지 않게 하여 속도감을 해치지 않는다.
- **모바일 네이티브 느낌**: 향후 앱 배포를 고려해, 웹에서도 앱처럼 매끄러운 제스처·터치 반응·전면 레이아웃을 구현한다.
- **일관된 디자인 시스템**: 색·간격·폰트·컴포넌트를 규칙화하여 어느 화면을 봐도 하나의 브랜드로 읽히게 한다.
- **AI 티 안 나는 완성도**: 어색한 자동 생성 느낌(어긋난 정렬, 촌스러운 그라데이션 등)을 피하고, 사람이 정성껏 만든 듯한 마감.

## 화면 흐름 (3화면)

### 화면 1 — 시작 (영감 트레이)
- 한 줄 설명: "영감 사진을 올리면, 이달의 네일 아트 시안이 나와요"
- **영감 트레이**: 사진을 최대 3장까지 담는 슬롯. 갤러리 선택/카메라 촬영으로 추가, 슬롯별 삭제 가능. 사진을 추가하는 행위 자체는 횟수 차감 없음.
- 1장 이상 담기면 옵션 노출:
  - 손톱 모양: 아몬드 / 라운드 / 스퀘어 / 스틸레토 (아이콘 선택)
  - 길이: 짧게 / 중간 / 길게
- "네일 디자인 만들기" 버튼 → 담긴 사진 전체를 종합해 한 번에 생성

### 화면 2 — 생성 중
- 10~20초 대기 동안 단계별 문구 순환 표시 ("사진의 색감을 읽는 중...", "어울리는 파츠를 고르는 중...")

### 화면 3 — 결과 (진화 루프의 중심)
- **콜라주**: AI 생성 네일 이미지(손에 시술된 실사 느낌)를 크게, 그 위에 입력한 영감 사진들이 작은 사각형으로 얹힌 형태. 콜라주 합성은 클라이언트(캔버스)에서 수행 — AI 추가 비용 없음.
- 무드 키워드 2~3개 + 대표 색상 팔레트 (예: "청량한 아쿠아 · 글레이즈드")
- 버튼:
  - **이미지 저장** — 콜라주 한 장을 기기에 저장
  - **사진 더하고 진화시키기** — 화면 1의 트레이로 돌아가 사진 추가/교체 후 재생성 (핵심 동선)
  - **다시 생성** — 같은 사진 세트로 한 번 더
  - **새로 시작** — 트레이 비우고 처음부터
- 남은 횟수 표시 (예: "오늘 2회 남음")
- "다시 생성"·"진화" 모두 생성 1회로 차감된다 (매 생성마다 실제 비용이 발생하므로). 사진 장수는 차감과 무관 — "만들기" 클릭 1번 = 1회.

## 생성 파이프라인 (방법 A)

1. **클라이언트 리사이즈**: 업로드 사진 각각을 브라우저에서 긴 변 1024px 이하 JPEG로 축소 후 전송 (속도/비용 절약).
2. **서버 API 라우트에서 Gemini 이미지 모델 호출** (API 키는 서버에만 보관, 영감 사진 1~3장을 모두 입력으로 전달):
   - 지시문: 입력된 사진 **전체**의 색감·질감·분위기를 종합한 하나의 네일 디자인 (사진이 여러 장이면 각 사진의 무드가 모두 읽히도록) / 선택한 모양·길이 반영 / 실제 손에 시술된 실사 사진 스타일 / 이미지와 함께 무드 키워드 2~3개 + 대표 색상 텍스트 반환.
   - **응답 형식**: 이미지 1장 + 텍스트(무드 키워드·색상)를 한 응답에서 받는다. 텍스트는 JSON 형태로 요구해 파싱한다 (파싱 실패 시 텍스트 생략하고 이미지만 표시 — 생성 실패로 처리하지 않음).
   - **트렌드 반영**: 지시문에 현재 유행하는 네일 트렌드 키워드(시즌 스타일, 인기 파츠/기법)를 포함시킨다. 이 트렌드 키워드는 코드에 하드코딩하지 않고 환경 변수 또는 설정 파일로 분리하여, 트렌드가 바뀔 때마다 주기적으로(예: 시즌별) 손쉽게 갱신할 수 있게 한다.
3. **클라이언트 콜라주 합성**: 생성된 네일 이미지를 바탕으로, 입력한 영감 사진들을 작은 사각형으로 겹쳐 얹은 콜라주를 캔버스로 합성 (`ref/` 형식). 영감 사진 위치는 네일이 가려지지 않게 가장자리 배치, 장수(1~3장)에 따라 프리셋 레이아웃 사용.
4. **응답 표시**: 콜라주 + 설명 텍스트를 화면 3에 렌더링. 저장 버튼은 콜라주 캔버스를 이미지로 내보낸다.

## 네일 미감 기준 (AI 지시문의 스타일 앵커)

`ref/` 폴더 11장이 목표 미감이다. 지시문 작성 시 아래 특징을 반영한다:

- **광택·질감**: 글레이즈드(유리알) 광택, 크롬/마그네틱 반짝임, 시럽·젤 특유의 투명 레이어감
- **파츠**: 입체(3D) 파츠 — 꽃, 불가사리, 진주, 나비, 리본 등 영감 사진에서 연상되는 오브제
- **톤**: 몽환적이고 채도를 절제한 K-네일 감성 (원색 떡칠 금지)
- **구도**: 실제 손에 시술된 모습, 자연스러운 손 포즈, 네일이 주인공인 클로즈업
- 검증 시 생성 결과를 `ref/` 이미지들과 나란히 놓고 "같은 디자이너 계열로 보이는가"를 판단 기준으로 삼는다.

## 횟수 제한 (남용 방지)

- 서버 측에서 IP 기준(Vercel `x-forwarded-for`) 1일 3회 카운트, 자정(KST) 리셋. Upstash Redis(무료 플랜)에 기록.
- 브라우저 저장소만으로는 시크릿 창 우회가 가능하므로 서버 측 카운트가 필수.
- 서비스 전체 하루 총량 제한 200회(환경 변수로 조정 가능)를 추가로 두어 비용 폭주 방지.
- 생성 실패 시 횟수 차감하지 않음. 차감 기준은 "만들기 클릭 1번 = 1회"이며 사진 장수와 무관.
- **남은 횟수 조회**: 생성 API 응답에 남은 횟수를 함께 내려주고, 첫 진입 시엔 가벼운 조회 API(또는 같은 라우트의 GET)로 확인해 화면에 표시.
- 알려진 한계(수용): 같은 네일샵 와이파이를 쓰는 여러 디자이너가 동일 IP로 묶여 서로 횟수를 소모할 수 있다. MVP에서는 수용하고, 문제가 되면 다음 단계에서 기기 식별을 보완한다.

## 기술 스택

- **프레임워크**: Next.js (프런트 + API 라우트 통합)
- **배포**: Vercel 무료 플랜
- **횟수 저장소**: Upstash Redis 무료 플랜
- **AI**: Google Gemini 이미지 생성 API (서버 측 호출만)
  - **기본 모델**: Nano Banana 2 (`gemini-3.1-flash-image`) — 참고 이미지 최대 14장 입력 지원, 장당 약 $0.045~0.067(1K). 하루 총량 200회를 다 써도 최대 ~$13/일.
  - **승격 경로**: `ref/` 비교 검증에서 품질이 부족하면 Nano Banana Pro(`gemini-3-pro-image`, 장당 ~$0.134)로 교체. 모델 ID는 환경 변수로 분리해 한 줄 교체 가능하게 한다.
  - Lite(`gemini-3.1-flash-lite-image`)는 이미지 입력이 안 되므로 사용 불가.
- **DB/인증**: 없음

## 검증 방법

- `ref/`의 실제 콜라주 11장에서 영감 사진 부분(작은 사각형들)을 잘라 입력으로 넣어, 원본 디자이너의 네일과 유사한 방향(색감·파츠·무드)의 결과가 나오는지 눈으로 비교 검증. → **디자이너가 실제로 아이디어 출발점으로 쓸 만한 수준인지**가 핵심 판단 기준.
- **진화 검증**: 사진 1장으로 생성 → 성격이 다른 사진 1장을 추가해 재생성했을 때, 결과에 두 사진의 무드가 모두 읽히는지 확인 (예: 빨강 단독 vs 빨강+초록).
- 결과 시안이 최신 트렌드를 반영하는지, 촌스럽거나 AI 티 나는 느낌은 아닌지 `ref/`와 나란히 놓고 디자이너 눈높이에서 확인.
- 콜라주 합성: 영감 사진 1·2·3장 각각의 레이아웃이 자연스럽고 네일을 가리지 않는지 확인.
- 모바일 화면(iPhone 크기)에서 업로드 → 생성 → 콜라주 저장 전체 흐름 동작 확인 (iOS Safari의 이미지 저장 동작 포함).
- 횟수 제한: 4회째 시도 시 차단되는지, 실패 시 차감 안 되는지 확인.

## 수요 검증 지표 (성공을 뭘로 판단하나)

회원가입·DB 없이도 측정 가능한 최소 지표를 Upstash Redis 카운터로 기록한다 (개인정보 없음, 숫자만):

- **일별 생성 수** — 횟수 제한 카운트로 이미 존재
- **저장 클릭 수** — 결과가 "쓸 만했다"는 가장 강한 신호. 생성 수 대비 저장 비율을 본다.
- **진화 사용 수** — "사진 더하고 진화시키기" 클릭 수. 핵심 기능이 실제로 쓰이는지 확인.
- 방문·재방문은 Vercel Analytics(무료)로 확인.

판단 기준(가설): 초기 사용자 중 저장 비율이 낮으면 결과 품질 문제, 진화 사용이 없으면 기능 매력 문제로 해석하고 방향을 조정한다.

## 범위 제외 (이번 MVP에서 안 하는 것)

- 회원가입/로그인, 결제, 생성 이력 보관 (진화는 세션 안에서만 — 브라우저를 닫으면 트레이가 비워져도 무방)
- 네일샵 예약 연동, 네일리스트용 고급 기능(포트폴리오 관리 등)
- **iOS/Android 네이티브 앱** (방향성만 확정, 실제 앱 개발은 웹 검증 이후 단계)
- 다국어 (한국어만)
- 4장 이상 사진 입력, 콜라주 레이아웃 수동 편집(위치/크기 조절), 이미지 보정 기능
- 손님용 확장 (디자이너 검증 이후 고려)
